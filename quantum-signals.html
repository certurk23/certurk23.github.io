▶ Methodology & Column Definitions

1. Decision (BUY / HOLD)

Decision is produced by a pure signal confluence rule. The system computes a count of active technical signals on the latest daily bar for each symbol.

  • BUY = Signal count ≥ confluence threshold (currently 4)
  • HOLD = Signal count < confluence threshold

No RSI condition is used in the decision rule. RSI is displayed for reference only.
No tier condition is used. Tier does not affect the BUY/HOLD decision.
No regime / market filter is applied at decision time. The optimizer evaluated filter groups during training but the final strategy uses signal-only entries because applying all 38 filters simultaneously eliminates virtually all opportunities.

The signal family consists of: RSI oversold, MACD crossover, Bollinger Band breakout, moving average crossovers (SMA/EMA), volume breakout, Stochastic oversold, OBV trend, price momentum, mean-reversion, and composite signals — totaling 93 binary signals per bar.

2. WR% & BT Return% (Per-Symbol Backtest Metrics)

These columns show per-symbol backtest results, not portfolio-level or rolling metrics.

  • WR% = Win rate of that specific symbol over the full 2000–2026 history.
  • BT Return% = Cumulative compounded return of all trades on that symbol over the same period.

Each symbol is tested with 6 strategy variants. The variant producing the highest composite score is shown. These are in-sample historical metrics, not forward-looking predictions. Transaction costs of 0.05% per round-trip are included. Minimum 5 trades required.

3. Entry / Stop / Target

These are theoretical percentage-based levels calculated from the last closing price.

  • Entry = Last close price (reference level, not a limit order).
  • Stop = Entry × (1 − stop_pct). Fixed percentage stop-loss, not ATR-based.
  • Target = Entry × (1 + target_pct). Fixed percentage take-profit, not ATR-based.

These are model-derived exit rules, not live order levels. If neither stop nor target is hit within the hold period, the position is closed at market.

4. Tier (S / A / B / C)

Tier represents a composite historical robustness ranking (quartile-based).

Composite score formula:

  • min(return, 500%) × 0.30 — Capped return
  • win_rate × 0.25 — Consistency
  • min(PF, 10) × 10 × 0.25 — Risk-adjusted edge
  • max(0, 100 − DD) × 0.20 — Drawdown penalty
  • Multiplied by min(n_trades/20, 1.0) — Trade count penalty
  • S-Tier = Top 25% — historically strongest edge
  • A-Tier = 50th–75th percentile
  • B-Tier = 25th–50th percentile
  • C-Tier = Bottom 25%

Tier does NOT equal a recommendation. It only reflects historical response to the signal family.
